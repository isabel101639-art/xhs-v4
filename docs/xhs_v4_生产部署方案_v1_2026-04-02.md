## 3. 运行方式

在生产机上进入项目目录后，推荐使用：

```bash
docker compose up -d --build
```

查看运行状态：

```bash
docker compose ps
docker compose logs -f web
docker compose logs -f worker
docker compose logs -f beat
docker compose logs -f nginx
```

健康检查地址：

```text
http://<your-host>/healthz
```

## 5. 当前配置特性

当前应用已经支持：

1. 通过 `DATABASE_URL` 切换数据库
2. 通过 `SECRET_KEY` 切换会话密钥
3. 通过 `ADMIN_USERNAME` / `ADMIN_PASSWORD` 切换后台登录账号
4. 通过 `DEFAULT_TOPIC_QUOTA` 控制默认名额
5. 通过 `DEEPSEEK_API_URL` 切换 AI 接口地址
6. 通过 `healthz` 做健康检查

## 5.1 Zeabur Worker 建议

在 Zeabur 中建议再新增一个服务：

1. 服务名称：`xhs-v4-worker`
2. 代码来源：与 `xhs-v4` 相同的 GitHub 仓库
3. 启动命令：`./docker/entrypoint-worker.sh`
4. 需要复用 `xhs-v4` 的同一套环境变量，至少包括：
   - `DATABASE_URL`
   - `REDIS_URL`
   - `CELERY_BROKER_URL`
   - `CELERY_RESULT_BACKEND`
   - `SECRET_KEY`

当前项目内已支持：

1. Worker 联通检查任务
2. 异步生成候选话题任务
3. 异步热点抓取骨架任务
4. 异步图片生成任务
5. 任务状态查询接口

## 5.2 Zeabur Beat 建议

如果要启用后台自动调度，再新增一个服务：

1. 服务名称：`xhs-v4-beat`
2. 代码来源：与 `xhs-v4` 相同的 GitHub 仓库
3. 启动命令：`./docker/entrypoint-beat.sh`
4. 需要复用：
   - `DATABASE_URL`
   - `REDIS_URL`
   - `CELERY_BROKER_URL`
   - `CELERY_RESULT_BACKEND`
   - `SECRET_KEY`
   - `ENABLE_AUTOMATION_BEAT`

当前 `beat` 会每分钟执行一次自动调度检查，并根据后台的调度配置决定是否派发任务。

### 图片任务环境变量

如果你后续要启用真实文生图，而不是 SVG 兜底，需要再补以下变量：

1. `ASSET_IMAGE_PROVIDER`
2. `ASSET_IMAGE_API_URL`
3. `ASSET_IMAGE_API_KEY`
4. `ASSET_IMAGE_MODEL`
5. `ASSET_IMAGE_SIZE`

如果这些变量不填，当前系统仍可工作，但图片任务会自动降级为 SVG 配图兜底。

当前图片提供方建议值可选：

1. `svg_fallback`
2. `volcengine_ark`
3. `volcengine_las`
4. `openai`
5. `openai_compatible`
6. `generic_json`

除 `ASSET_IMAGE_API_KEY` 这类敏感项外，其他图片与热点默认参数现在也可以在后台 `自动化中心` 中维护。

如果使用火山方舟，建议补：

1. `ASSET_IMAGE_PROVIDER=volcengine_ark` 或 `volcengine_las`
2. `ASSET_IMAGE_API_BASE`
3. `ARK_API_KEY` 或 `LAS_API_KEY`
4. `ASSET_IMAGE_MODEL`

当前系统还支持按图片类型生成：

1. `medical_science`
2. `poster`
3. `checklist`
4. `memo`
5. `knowledge_card`
6. `myth_compare`
7. `flowchart`

当前自动化中心已支持：

1. 运维配置维护
2. 请求预览
3. 任务列表筛选
4. 任务详情查看
5. 运行诊断
6. 图片资产库管理
7. 产品图库 / 内容素材库 / 生成资产库分开维护
8. 图片生成按类型选择，例如医学科普型、知识卡片型、清单、备忘录、大字报
9. 热点数据源模板化导入与导入预览

后台管理页当前已支持：

1. 活动快照查看
2. 手动备份记录查看
3. 手动创建备份
4. 从备份记录恢复新活动

## 6. 数据库说明

### 开发环境

当前本地仍可继续使用：

1. `SQLite`

### 生产环境

建议使用：

1. `PostgreSQL`

### 注意事项

当前项目已经具备自动建表和基础字段迁移逻辑，但：

1. `SQLite -> PostgreSQL` 的历史数据迁移脚本还没有单独写。
2. 如果你要把现有本地数据带到正式环境，后续还需要补一版迁移方案。

## 7. 生产建议

1. 正式环境不要继续直接使用本地 `.env` 中的真实密钥。
2. 正式环境应通过服务器环境变量或受控的部署变量注入密钥。
3. `SESSION_COOKIE_SECURE` 在 HTTPS 环境下建议改为 `true`。
4. PostgreSQL 和 Redis 建议挂载独立数据卷，并做定期备份。
5. Nginx 前面如果再接云负载均衡，也建议保留 `healthz` 探针。
