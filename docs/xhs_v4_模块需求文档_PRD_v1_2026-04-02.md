- 文档版本：`v1.0`

- 适用项目：`xhs_v4`
- 文档状态：`需求整理稿`
- 产品定位：`肝病健康内容运营与任务管理平台`

## 1. 项目背景

当前 `xhs_v4` 已具备基础的报名、话题管理、AI 文案生成、多平台数据提交、数据分析、自动化中心等能力，但整体仍偏工具化。  
项目下一阶段目标是将其升级为一个可持续运营的网站型平台，覆盖以下完整链路：

1. 热点抓取与话题发现
2. 语料沉淀与爆款分析
3. 话题生成、审核、发布
4. 报名、创作、提报、补录
5. AI 文案与图片生成
6. 全局数据分析与个人账号表现分析
7. 每期任务隔离、历史归档、备份与复用

## 2. 产品目标

### 2.1 业务目标

1. 搭建可长期使用的网站化运营平台，而非单次任务工具。
2. 每月稳定生成 `80` 个可发布话题，每个话题名额支持按期数、按批次、按单个话题灵活调整，系统仅保留默认值。
3. 建立肝病相关的热点池、语料库、话题池和图库。
4. 提升内容生产效率，降低人工找题、拆题、写文案、做图的时间成本。
5. 建立以期数为单位的完整数据闭环，支持回溯、复盘、归档和复用。
6. 支持对报名人整体表现和个人账号表现进行追踪、分析和建议输出。

### 2.2 产品目标形态

平台需同时具备两种属性：

1. `前台网站`：给报名人查看话题、报名、生成文案、提交数据、查看个人建议。
2. `后台运营中台`：给管理员管理活动期数、储备内容、审核发布、分析数据、生成报告。

## 3. 用户角色

| 角色 | 说明 | 主要权限 |
|---|---|---|
| 超级管理员 | 系统负责人 | 全部权限、配置、发布、备份恢复 |
| 内容运营 | 负责热点、语料、话题、图片、发布管理 | 管理储备池、候选池、发布池 |
| 审核人员 | 负责审核话题、文案、图片、合规 | 审核通过/驳回 |
| 数据查看者 | 只看分析数据与报表 | 查看看板与导出 |
| 报名人 | 参与报名、生成内容、提交数据 | 查看话题、报名、提交、查看个人分析 |

## 4. 核心业务原则

1. `当期内容与储备内容彻底隔离`
   当期已发布话题、名额、数据、图片、语料快照，不受后台储备池调整影响。

2. `自动化内容必须经过人工审核`
   热点抓取结果、自动生成话题、自动生成文案、自动生成图片，默认进入候选区，不直接进入正式发布区。

3. `期数维度强管理`
   每一期活动都应具备独立的话题快照、数据快照、分析快照和归档能力。

4. `数据来源可追溯`
   所有热点、语料、图片、话题、个人账号数据，必须记录来源、时间、渠道、导入方式和处理状态。

5. `合规优先`
   涉及医疗健康内容、药品表述、爆款拆解、AI 文案与图片生成时，必须接入合规检测规则。

## 6. 模块详细需求

## M01. 网站门户与视觉系统

### 目标

将现有系统升级为完整网站，可由后台实时更新内容，整体风格支持：

1. `国际化专业风格`
2. `小红书内容平台风格`
3. 后台与前台可使用不同主题

### 功能需求

1. 首页支持活动期数展示、核心导航、最新话题、最新热点、最新公告。
2. 前台支持话题广场、报名中心、我的任务、个人分析等入口。
3. 后台支持运营工作台、数据中心、自动化中心、审核中心。
4. 支持主题切换、品牌色配置、导航配置、首页模块排序。
5. 所有页面需适配桌面端和移动端。

### 管理需求

1. 首页文案、Banner、导航菜单、活动入口可后台配置。
2. 支持每期独立封面文案和风格设置。
3. 支持网站公告、活动说明、报名须知的后台编辑。

### 验收标准

1. 不改代码也可以更新网站主要内容。
2. 前后台风格统一，且移动端可正常使用。
3. 同一系统可切换不同视觉主题。

## M02. 期数管理与内容隔离

### 目标

确保每一期活动独立管理，储备内容的调整不影响当期任务。

### 功能需求

1. 支持创建活动期数，包含名称、标题、描述、开始时间、结束时间、状态。
2. 每期支持独立管理：
   - 话题池
   - 名额
   - 报名数据
   - 提报数据
   - 图片与素材
   - 数据分析
   - 周报与归档
3. 支持 `储备池 -> 候选池 -> 当期正式池` 三层内容体系。
4. 当期发布后生成快照版本，后续修改储备内容不影响当期。
5. 支持历史期数调用与查看，不影响当前期。

### 数据规则

1. 每期默认生成 `80` 个话题。
2. 每个话题名额支持灵活配置，可按活动期、批量生成或单个话题分别设置。
3. 期数关闭后自动进入归档状态。
4. 归档期数支持只读查看、报表导出、再次复制为新一期模板。

### 验收标准

1. 后台调整储备语料、候选话题时，不影响当前已发布期数。
2. 任意历史期数都可以独立调出查看其话题和数据。

## M03. 话题广场与报名中心

### 目标

为报名人提供清晰的话题浏览、报名和查看入口。

### 功能需求

1. 前台展示当期已发布话题。
2. 每个话题展示：
   - 话题标题
   - 关键词
   - 撰写方向
   - 参考案例
   - 爆款链接
   - 剩余名额
3. 支持按分类、产品线、内容类型、热点来源筛选话题。
4. 报名时支持手机号识别历史账号信息。
5. 支持查看我的报名记录、状态、对应话题。

### 规则需求

1. 报满后不可继续报名。
2. 每个报名记录必须归属某一期、某个话题、某个组别。
3. 支持管理员手动调整名额。

### 验收标准

1. 用户可在前台顺利浏览话题并完成报名。
2. 可准确查看每个话题报名人数与剩余名额。

## M04. 多平台提报与更新中心

### 目标

支持报名人对不同平台的发布结果进行分步提交、补录和更新。

### 功能需求

1. 支持以下平台数据提报：
   - 小红书
   - 抖音
   - 视频号
   - 微博
2. 支持提交字段：
   - 发布链接
   - 曝光量/传播量
   - 阅读量
   - 点赞
   - 收藏
   - 评论
   - 笔记标题
   - 笔记正文
3. 支持先提交任一平台，后续补提其他平台。
4. 支持修改链接和指标。
5. 自动识别内容类型，用于分析。

### 规则需求

1. 任一平台有链接即可算有效发布。
2. 更新数据不应覆盖其他平台已有正确数据。
3. 所有更新动作必须记录更新时间与操作人。

### 验收标准

1. 同一报名记录可以多次补充平台数据。
2. 数据分析页可正确统计各平台表现。

## M05. AI 文案中心

### 目标

根据话题、关键词、提示词、人设，稳定生成更像真人的小红书爆款文案。

### 功能需求

1. 支持两种生成方式：
   - 有提示词生成
   - 无提示词自动生成
2. 支持一次生成 `3` 版文案。
3. 每版文案应包含：
   - 标题
   - 正文
   - 关键词
   - 推荐人设
   - 内容类型
4. 支持多种人设：
   - 患者本人
   - 家属视角
   - 体检人群
   - 职场应酬族
   - 医学助理
   - 病友经验视角
5. 支持真人化重写、合规检查、复制、收藏、再次生成。

### 优化要求

1. 去 AI 腔。
2. 去同质化。
3. 千人千面。
4. 带真实场景感和情绪感。
5. 符合医疗内容合规要求。
6. 与话题强相关，不跑题。

### 技术要求

建议生成链路分层：

1. 话题理解
2. 人设设定
3. 文案初稿生成
4. 去 AI 化处理
5. 相似度检测与反重复
6. 合规修正

### 验收标准

1. 同一话题连续生成的文案相似度可控。
2. 用户可直接复制和使用。
3. 生成内容可被后续分析模块识别与归类。

## M06. 图片库与文生图中心

### 目标

建立独立的图片资产能力，支持产品图管理和内容图生成。

### 功能拆分

#### A. 产品图库

1. 存储产品图、包装图、规范物料图。
2. 支持标签：
   - 产品名
   - 场景
   - 尺寸
   - 版权状态
   - 可用范围

#### B. 内容素材库

1. 医学科普图
2. 知识卡片
3. 检查流程图
4. 大字报
5. 误区对照图
6. 封面模板

#### C. 文生图任务中心

1. 输入话题、关键词、风格、用途生成图片。
2. 保存 prompt、生成结果、采用状态、关联话题。
3. 支持同一话题多版图片。

### 规则需求

1. 产品图库与内容图库必须分开管理。
2. 当期采用图片需快照保存，不受后续替换影响。

### 验收标准

1. 能为每个话题提供可选图片素材。
2. 能按期数、话题、产品线追溯图片来源。

## M07. 热点抓取与选题引擎

### 目标

自动抓取与肝病相关的高搜索、高热度话题，形成可评分的热点池，并据此生成候选话题。

### 数据源范围

1. 小红书
2. 抖音
3. 千瓜
4. 其他后续可扩展平台

### 功能需求

1. 定时抓取热点数据。
2. 存储原始抓取结果与清洗结果。
3. 对热点进行评分。
4. 提取关键词、爆款标题、爆款链接、作者、互动、搜索热度。
5. 基于热点自动生成当月 `80` 个候选话题。
6. 每个候选话题需携带可调整的名额配置，生成时可批量指定，审核前后也可单独修改。
7. 每个候选话题需包含：
   - 话题标题
   - 关键词
   - 热度分
   - 推荐内容类型
   - 推荐人设
   - 软植入方向
   - 参考爆款链接
   - 撰写案例

### 热点评分建议

建议综合以下维度：

1. 搜索热度
2. 点赞
3. 收藏
4. 评论
5. 时间新鲜度
6. 肝病相关度
7. 是否适合产品软植入
8. 重复度

### 验收标准

1. 管理员可查看热点池和候选话题池。
2. 话题生成结果可经审核后发布到话题广场。

## M08. 语料库与爆款分析中心

### 目标

自动沉淀肝病相关爆款内容、产品内容、模板内容，形成可持续更新的语料库。

### 语料来源

1. 爆款笔记抓取
2. 产品规范内容
3. 历史优质提报内容
4. 人工录入内容
5. 第三方数据源导入内容

### 语料类型

1. 爆款拆解
2. 医学科普
3. 产品卖点
4. 合规表达
5. 封面模板
6. 开头钩子模板
7. 结尾互动模板
8. 图片模板

### 功能需求

1. 自动抓取爆款内容并入库。
2. 自动提取：
   - 标题结构
   - 开头钩子
   - 内容结构
   - 情绪类型
   - 人设类型
   - 关键词埋点
   - 互动结尾
3. 自动生成：
   - 模板
   - 语料摘要
   - 提示词
   - 可复用文案结构
4. 支持搜索、标签、筛选、使用次数统计。

### 验收标准

1. 语料库不是纯内容堆积，而是可搜索、可分析、可复用。
2. AI 文案中心与话题引擎可以调用语料库。

## M09. 自动化中心与审核发布流

### 目标

建立完整的自动化运营链路，但发布必须经过人工确认。

### 功能需求

1. 统一展示：
   - 语料数量
   - 热点数量
   - 候选话题数量
   - 已发布话题数量
2. 支持操作：
   - 导入热点
   - 管理语料
   - 生成候选话题
   - 查看提示词
   - 审核通过/驳回
   - 发布到指定期数
3. 发布到话题广场前，必须由管理员确认。

### 状态流建议

1. 草稿
2. 待审核
3. 审核通过
4. 已发布
5. 已归档
6. 已驳回

### 验收标准

1. 自动抓取和自动生成不会直接污染当期正式内容。
2. 管理员可明确掌控发布动作。

## M10. 全局数据分析中心

### 目标

形成可实时刷新的活动级运营看板，并输出下一期建议。

### 核心看板指标

1. 报名人数
2. 发布人数
3. 发布条数
4. 各平台发布数
5. 各平台互动、阅读、曝光表现
6. 小组数据
7. 个人数据
8. 内容类型表现
9. 热点词趋势
10. 下一期选题建议

### 分析维度

1. 按活动期
2. 按时间区间
3. 按平台
4. 按小组
5. 按个人
6. 按内容类型
7. 按话题
8. 按人设

### 输出能力

1. 周报导出
2. 活动复盘报告
3. 优秀个人榜
4. 小组排名
5. 内容优化建议
6. 下期选题建议

### 个人分析能力

每位报名人应能看到：

1. 自己报名了哪些话题
2. 已发布多少条
3. 各平台表现
4. 最佳内容类型
5. 下一步建议

### 验收标准

1. 数据看板可按期数和时间实时刷新。
2. 可稳定输出汇报级别报表。

## M11. 报名人账号数据看板

### 目标

新增独立模块，对所有报名人的账号维度表现进行持续追踪和分析。

### 模块定位

该模块不是“单条提交数据统计”，而是“账号经营结果看板”。

### 核心对象

每位报名人对应至少一个账号主体：

1. 小红书账号
2. 抖音账号
3. 其他平台账号可扩展

### 核心功能

1. 建立报名人与账号的绑定关系。
2. 追踪账号在指定周期内的发布情况。
3. 统计账号级数据：
   - 发布话题数
   - 发布笔记数
   - 涨粉情况
   - 阅读量
   - 曝光量
   - 点赞
   - 收藏
   - 评论
   - 互动率
4. 识别：
   - 效果最好的一条笔记
   - 爆款笔记
   - 连续表现较弱的笔记
5. 输出账号级建议：
   - 更适合的内容类型
   - 更适合的人设
   - 更适合的话题方向
   - 更适合的发布频次

### 页面形态建议

#### A. 报名人总览页

展示所有报名人的账号表现列表，字段包括：

1. 姓名
2. 组别
3. 小红书账号
4. 当前期报名数
5. 当前期发布数
6. 当前期总阅读
7. 当前期总互动
8. 涨粉数
9. 爆款笔记数
10. 最佳笔记标题

#### B. 报名人详情页

展示单个报名人的账号经营全貌：

1. 基础信息
2. 历史报名记录
3. 历史发布记录
4. 粉丝趋势
5. 单条笔记表现趋势
6. 话题命中情况
7. 爆款笔记列表
8. AI 建议

### 数据来源策略

该模块建议采用 `三层采集模式`：

#### 方案 A. 官方授权采集

适用于有官方开放接口的平台。  
授权后可采集账号公开信息、作品列表、作品数据等。

#### 方案 B. 第三方数据源接入

适用于无法直接通过官方开放接口稳定获取创作者内容数据的平台。  
例如企业采购后的第三方内容数据平台。

#### 方案 C. 人工导入兜底

支持通过 Excel、CSV、接口回传、手工录入补充账号数据，保证平台不可用时仍能运转。

### 规则需求

1. 每位报名人可绑定多个账号，但需明确主账号。
2. 涨粉需按时间周期计算，不可只存静态当前粉丝数。
3. 爆款笔记规则应后台可配置，例如：
   - 阅读量阈值
   - 收藏阈值
   - 互动率阈值
4. 账号级分析不得直接覆盖报名人手动提交的数据，应单独存储。

### 风险提示

1. 小红书等平台的账号数据采集能力受授权、开放能力、第三方采购和平台策略影响。
2. 不建议将该模块设计为“纯爬虫强依赖”模式，应优先采用 `授权 + 第三方数据 + 人工导入` 的混合模式。

### 验收标准

1. 可查看每位报名人的账号经营数据趋势。
2. 可识别每位报名人的最佳笔记和爆款笔记。
3. 可输出个人运营建议。

## M12. 报表、备份与恢复中心

### 目标

支持每期报表导出、数据归档、自动备份与一键恢复。

### 功能需求

1. 导出周报、月报、活动复盘。
2. 导出原始报名、提报、话题、账号表现数据。
3. 支持活动归档。
4. 支持数据库备份与文件备份。
5. 支持按期恢复。
6. 支持复制历史期为新一期模板。

### 验收标准

1. 任意已归档期数可追溯查看。
2. 数据误操作后可恢复。

## M13. 权限、日志与系统配置

### 目标

保证系统长期运行时的安全、可管理、可审计。

### 功能需求

1. 用户角色权限控制。
2. 所有关键动作写入操作日志：
   - 导入热点
   - 编辑语料
   - 生成话题
   - 审核发布
   - 修改名额
   - 修改数据
   - 归档
   - 恢复
3. 定时任务中心：
   - 热点抓取任务
   - 语料更新任务
   - 账号数据同步任务
   - 图片生成任务
4. 数据源配置中心：
   - 小红书
   - 抖音
   - 千瓜
   - 其他接口

### 验收标准

1. 关键动作可追溯到人和时间。
2. 定时任务可监控、可重试、可暂停。

## 10. 结论

`xhs_v4` 下一阶段不应继续被定义为“报名工具”，而应定义为：

`肝病健康内容运营平台 + 任务管理平台 + AI 内容生产平台 + 数据复盘平台`

其中新增的“报名人账号数据看板”模块具备明确业务价值，建议纳入二期重点建设。  
但在技术路线设计上，应优先采用 `授权接入 + 第三方数据源 + 人工导入兜底` 的模式，以降低对单一平台抓取能力的依赖。
