# xhs_v4 项目状态总览

- 版本：`v1`
- 更新日期：`2026-04-10`
- 适用范围：`xhs_v4 当前本地仓库状态`

## 1. 当前判断

截至 `2026-04-10`，`xhs_v4` 已经从“报名工具”发展成一套可运营的内容运营平台骨架。

当前建议判断为：

1. `P0 可运营版本`：已基本成型，业务主链路已经跑通
2. `P1 增强版本`：已进入实施阶段，但还没有全部收口
3. `P2 智能运营版本`：尚未真正展开

综合估算完成度可按 `70%~75%` 理解。

## 2. 已完成较多的部分

1. 网站门户与视觉系统
2. 活动期管理、发布、归档、复制、快照恢复
3. 话题广场、话题详情、报名、我的报名
4. 多平台提报与更新字段
5. AI 文案生成、真人化、合规检查、图文创作包
6. 数据分析页、周报、月报、活动复盘导出
7. 后台管理台基础框架
8. 门户配置、公告管理
9. 备份恢复、活动快照
10. 管理员、角色权限、系统设置、操作日志

## 3. 已有结构但还没完全收口的部分

1. 图片库与文生图中心
当前已有任务流、素材库、样式系统，但默认仍以 `SVG fallback` 为主，真实图片模型还需接入。

2. 热点抓取与选题引擎
当前已有手动导入、模板预览、Worker 骨架任务，但真实热点源还没完全接稳。

3. 自动化中心与审核发布流
当前已有候选话题池、审核、发布、调度配置，但生产级 Worker / Beat / Celery 链路还需补齐。

4. 报名人账号数据看板
当前已有后台页面和接口，但样本数据不足，真实分析能力还需靠运营数据验证。

5. 语料库与爆款分析中心
当前语料库基础已经有，爆款分析和账号联动分析仍需更多真实数据支撑。

## 4. 当前未完成的关键项

1. 真实热点源接入并稳定产出热点池
2. 真实图片模型接入，替换 `SVG` 兜底
3. `Redis / Celery Broker / Celery Backend / Worker / Beat` 生产联通
4. 候选话题池首批真实记录和审核发布闭环验证
5. 账号、笔记、快照等样本数据灌入
6. `app.py` 模块化拆分

## 5. 当前最值得继续做的事

建议按下面顺序继续推进：

1. 先补齐异步运行环境
目标：让自动化中心从“可展示”进入“可实际运行”

2. 接入真实热点源
目标：让热点池、候选话题、审核发布流有真实输入

3. 接入真实图片模型
目标：让图片中心从演示态进入可运营态

4. 导入账号与笔记样本数据
目标：让账号看板和分析中心真正有数据可看

5. 开始拆分 `app.py`
目标：降低后续继续开发和回归修改的风险

## 6. 新增查看入口

这次已新增后台项目状态页，后续可直接从以下入口查看：

1. 后台固定入口：`/admin/project-status`
2. 后台标签页入口：`/admin?tab=status`
3. 状态页支持“补齐演示数据”，可一键为热点池、候选话题池、账号看板、图片中心补演示记录，用于联调和展示

## 7. 备注

本总览是对当前仓库代码、页面、接口、数据库状态与本地冒烟结果的综合判断，不是外部发布版本说明。
